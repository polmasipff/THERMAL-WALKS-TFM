# `notebooks/` — analysis record

Curated, output-stripped notebooks documenting the analysis, organised by the
thesis chapter they feed. Each chapter folder also contains the local helper
modules (`*.py`) its notebooks import, so imports resolve when you launch
Jupyter **from inside the notebook's own folder** (the default working
directory). Data is read from `../../data/` (see `data/README.md`).

```
01_effective_temperature/   Construction of T_corr, f_sun/f_mix/wind_sc, T_env, HDX  (Ch 2–3)
02_state_space_ch3/         Separability, grouping diagnostics, C/N/U choice, ML     (Ch 3 / App A)
03_blume_capel_ch4/         Spin-1 / Blume-Capel reading: m(T), q(T), fields         (Ch 4)
04_transition_dynamics_ch5/ Stop-to-stop transitions, Markovianity, regimes, memory (Ch 5)
05_exploratory/             TCV/TSV scatters, distributions, T–HDX                   (exploratory)
```

## Running them

1. `pip install -r ../requirements.txt`
2. Put the input data in `../data/` (see `data/README.md`).
3. Launch Jupyter **from the notebook's folder** so that `../../data/` resolves
   and the sibling helper modules import:
   ```bash
   cd notebooks/04_transition_dynamics_ch5
   jupyter lab Non-eq-analysis-Tenv.ipynb
   ```

## What is verified vs. what is a record

- The **figure scripts in `../figures/`** are the verified, turnkey way to
  reproduce the thesis figures from the canonical datasets.
- These notebooks are the **authored analysis record**. Their *primary* data
  loads have been repointed to `../../data/` and all hard-coded personal paths
  removed. Some notebooks additionally read *pipeline-intermediate* CSVs
  (`saved_df/`, `markov_newT/`, `markov_results/`, `results_onset/`, `csvs/`)
  that are regenerated by running the upstream notebooks in order; those
  folders are git-ignored. The exploratory notebooks read the raw
  `all_surveys(votes).csv` from `../../data/`.
